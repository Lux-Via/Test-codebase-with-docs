# Test Codebase with Docs

A trial project for AI-powered documentation updates.

Every push to `main` triggers a GitHub Actions workflow that:
1. Reads the git diff (all source file types: `.py`, `.html`, `.css`, `.xml`, `.js`, etc.)
2. Calls GitHub Models (`gpt-4o-mini`) to update `/docs`
3. Syncs `/docs` to the GitHub Wiki automatically

## Structure

```
├── src/                   application source code
├── docs/                  documentation (auto-updated by AI on every push)
├── .github/
│   ├── workflows/
│   │   └── update-docs.yml    the CI/CD pipeline
│   └── scripts/
│       └── update_docs.py     the AI doc updater script
└── README.md
```

## Setup — GitHub Token (required once per developer)

The workflow uses two separate tokens:

| Token | What it does | Where it lives |
|---|---|---|
| `GITHUB_TOKEN` | Auto-generated by GitHub Actions — pushes docs commits, syncs wiki | Automatic, no setup |
| `MODELS_TOKEN` | Your personal PAT — authenticates the GitHub Models AI API call | Repo secret + local shell |

### Step 1 — Create a fine-grained PAT with Models access

1. Go to `github.com` → click your profile photo → **Settings**
2. Scroll to the bottom of the left sidebar → **Developer settings**
3. **Personal access tokens** → **Fine-grained tokens**
4. Click **Generate new token**
5. Fill in:
   - **Token name**: `github-models-workflow`
   - **Expiration**: 90 days (recommended — renew before it expires)
   - **Resource owner**: your personal account (e.g. `WeOnlyLiveOnce13`)
   - **Repository access**: All repositories
6. Under **Permissions** → click the **Account** tab
7. Click **+ Add permissions** → search for **Models** → set to **Read-only**
8. Click **Generate token** — confirm in the popup
9. **Copy the token immediately** — it is shown only once

> The `Models: Read-only` permission is under the **Account** tab, not the Repositories tab.
> It is only available on tokens owned by your personal account, not organisation-owned tokens.

---

### Step 2 — Add the token as a repository secret

This allows the GitHub Actions workflow to use your token without exposing it.

1. Go to `github.com/Lux-Via/Test-codebase-with-docs`
2. **Settings** → **Secrets and variables** → **Actions**
3. Click **New repository secret**
4. Name: `MODELS_TOKEN`
5. Value: paste the token you just copied
6. Click **Add secret**

To update an expired token later: click the pencil icon next to `MODELS_TOKEN` → paste the new token.

---

### Step 3 — Store the token locally (for local testing only)

The token stored here lets you call GitHub Models from your machine directly.
It is **not** required for the GitHub Actions workflow — that uses the repo secret.

**Add to `~/.zprofile`** (Mac — runs once at login, persists across terminal restarts):

```bash
# Open the file
open ~/.zprofile

# Add this line at the bottom — replace with your actual token
export GITHUB_TOKEN=github_pat_your_token_here
```

Save the file, then reload it in your current terminal:

```bash
source ~/.zprofile

# Verify it loaded
echo $GITHUB_TOKEN
```

> Use `~/.zprofile` (not `~/.zshrc`) — profile files run once at login and are the convention for secrets.

---

### Renewing an expired token

Tokens expire after 90 days. When that happens:

1. Go to **Developer settings** → **Fine-grained tokens** → delete the expired token
2. Follow Step 1 above to create a new one
3. Update the repo secret (Step 2) with the new token
4. Update `~/.zprofile` (Step 3) with the new token

## How the AI doc update works

```
git push to main
      ↓
GitHub Actions reads git diff (all source files except docs/ and .github/)
      ↓
Calls gpt-4o-mini via GitHub Models API (authenticated with MODELS_TOKEN)
      ↓
AI updates docs/overview.md, docs/api-reference.md, docs/changelog.md
  — attributes changelog entry to the pushing developer (@username)
  — preserves any manual doc edits in the same commit
      ↓
Bot commits updated docs back to main [skip ci]
      ↓
Syncs docs/*.md to GitHub Wiki
```

## Troubleshooting

| Error | Cause | Fix |
|---|---|---|
| `401 Unauthorized — models permission required` | `MODELS_TOKEN` secret missing or lacks `Models: Read` scope | Recreate token with Models: Read-only under Account permissions |
| `401 Unauthorized` on wiki clone | `GITHUB_TOKEN` permissions issue | Check repo Settings → Actions → Workflow permissions → set to Read and write |
| Wiki sync skipped | Wiki not initialised | Go to Wiki tab → create first page |
| No doc changes after push | Only non-source files changed | Push a `.py`, `.html`, `.css`, `.xml` etc. file to trigger the AI |
